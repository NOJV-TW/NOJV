     1.5.1     Binary and Hex
     Binary:
         Similar to decimal, when a digit reaches 2, it is carried over to the next
     digit.
     Example: 1(2) + 1(2) will be 10(2) because 2 is greater than 1 so it will be
     carried over to the next digit.
         All data in computers are binary. Each digit is called a “bit,”and 8
     bits make a “byte.”Therefore, a byte’s max value is 11111111(2) , which
     is 127 in decimal.
         Hex:
         By grouping every 4 digits of a binary number, we can convert it to 1
     hex digit represented by 0 ∼ 9 then A∼F (resembles 10 ∼ 16).
         Example 1: 1010(2) = 10(10) = A(16) .
         Example 2: 1010101111001101(2) = ABCD(16) .

1.5.2.2   Signed Integer
The first bit of a signed integer represents its sign, where 1 means negative,
and 0 means positive.
    Positive:
It works just like an unsigned integer.
    Negative:
The scheme for negative numbers is called ”two’s complement.” In this
scheme, to convert a positive number x into -x, we flip all the bits and
then add 1. The reasoning behind this will be taught in later courses.
    Note: Flip means reversing 1s and 0s.
    Example:
12 in binary is 0000000000001100(2) .
To turn 12 into −12, first:
flip(12) → 1111111111110011(2) .
Then add 1, which equals:
1111111111110100(2) .
Therefore, −12 in binary is 1111111111110100(2) .


1.5.2.3   Float
In binary, floats are stored in scientific notation using the following format:
S(1.F ) × 2EXP −15
Looking at a float in binary from left to right:
Bit 1: Sign bit (1 for negative, 0 for positive) (S)
Bits 2∼6: Exponent(EXP)
Bits 7∼16: Fraction(F)

   Example 1: F = 101, 1.F is 20 + 2−1 + 2−3 = 1.625
   Example 2: Given 0100011010000000(2) :


   • The first bit is 0, which means it’s a positive number.


   • EXP = 10001(2) = 17


   • F = 1010000000(2)


Therefore this binary represents:
1.101(2) × 2(17−15) = 1.101(2) × 22 = 2.625 × 22 = 6.5
Binary float 0100011010000000(2) is +6.5 in decimal.
