Course Experience
Redesign

15 screens · 1 aesthetic Spec: 2026-04-11-course-experience-redesign-design.md Branch: course-experience-redesign

Flat, visual reference for every route the course redesign touches. Open each file directly in a browser — no build step. Shared tokens live in shared.css and mirror apps/web/src/app.css.

How to use these. Each prototype is visual-only: click/hover states are decorative and forms don't submit. The point is to lock in typography, spacing, card density, and teacher/student variation before writing the Svelte implementation. When executing the plan, each UI task references the prototype file it maps to.
Section 1 · Courses entry
01 /courses
Courses listing

Two tabs — enrolled vs managing. Active/Archived filter chips. Role-aware card variants. Empty state included.

02 /courses/new
Create course

Two cards only — basics + default policies. No slug/semester/visibility. Late penalty as rule builder with custom %. No draft button.

Section 2 · Cross-course aggregated views
14 /assignments
Assignments (top-level)

All assignments across all my courses. Course tag per row. No create button, no Draft filter. Sorted by urgency.

15 /exams
Exams (top-level)

All exams across my courses. Running exams get live pulse + enter button. Course tag per row. No create affordance.

Section 3 · Course interior
03 /courses/[courseId]
Overview tab

Single-column stack: Announcements → Assignments → Exams. Full datetime on time ranges. Draft rows visible to teacher/TA.

12 /courses/[courseId]/members
Members

Bulk handle paste panel (teacher only). Role chips, search, roster rows with email/role controls. Placeholder user rows shown for unregistered.

13 /courses/[courseId]/settings
Settings

Stacked cards: course info, default policies, visibility, danger zone (delete with typed confirmation).

Section 4 · Course assignments
04 /courses/[courseId]/assignments
Assignments list

All/Open/Upcoming/Closed + Draft for teacher. No HW number prefix. Full datetime. Class stats right side. Draft rows dashed + visible to TA.

05 /courses/[courseId]/assignments/new
Create assignment

Basics / Problems / Schedule / Advanced. No upper bound on problems — can save empty draft for TAs to fill. Per-day attempts. Late penalty rule builder.

06 /courses/[courseId]/assignments/[id]
Assignment detail — student

Problem list with AC→light green row, partial→light red row. No status pills, no score estimation. Running submission log.

07 /courses/[courseId]/assignments/[id]
Assignment detail — teacher

Sub-tab strip: Problems | 提交 | 抄襲 | 設定. Each tab prototyped. Plagiarism right-drawer diff sketch.

Section 5 · Course exams
08 /courses/[courseId]/exams
Exams list

Upcoming/Running/Ended/Draft chips. Date block left side. Proctor icons inline. Live pulse for running.

09 /courses/[courseId]/exams/new
Create exam

Assignment form + Proctoring card (page lock / IP binding / whitelist) + Scoring card (ICPC/IOI, freeze, cooldown).

10 /courses/[courseId]/exams/[id]
Exam — registration view (State A)

Pre-start: schedule, proctoring summary, registered count. Student Start button disabled until start. Teacher sub-tabs visible.

11 /courses/[courseId]/exams/[id]/problems/[idx]
Exam mode — problem view

Uses same ProblemSolveView layout as /problems/[id]. Site nav replaced by exam strip (lock + countdown + exit). Submissions panel filtered to this exam only.
